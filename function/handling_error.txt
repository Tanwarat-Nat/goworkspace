reduce functionality :
ถ้าเกิด withdraw ไม่ได้จริงๆ แทนที่จะออกไป ก็ไม่ออกนะ ถึงจะ withdraw ไม่ได้
แต่คุณสามารถทำอย่างอื่นได้ คุณอาจจะ transfer ได้ หรือจ่ายเงินผ่าน QR ได้
หรือจะบอก user ว่ามี allternative อันอื่นอีกมั้ยให้เขาใช้

ignor error:
เป็น rare case มากๆ ในความเป็นจริงเราไม่ควร ignor error เราควรมอง error
เป็นทองคำ เป็นตัวที่บอกเราว่า เห้ยย โปรแกรมของเรามีปัญหานะ เราไม่ควรชะล่าใจ
เราควรจะเข้าไปดูตรงนั้นด้วยความละเอียดทีถ้วน

แต่ก็มีบางเคสที่เราสามารถ ignor ได้ ถ้าโปรแกรมของเราสร้าง temporary file ขึ้นมาอันนึง
ซึ่งแต่ละ os ก็จะต่างกัน window ก็จะสร้าง temp อีกทีนึง unix ก็จะสร้าง temp อีกทีนึง
แล้วโปรแกรมของเราเกิดทำการไป delete file ที่โปรแกรมนั้นสร้างขึ้นมาเอง ซึ่งสร้างอยู่
ใน temporary floder นั้น แต่ว่า os ดันไปลบออกไปซะก่อน หมายความว่า ในขณะที่
โปรแกรมของเรากำลังไปลบเนี่ย หึ้ยมันหาไฟล์ไม่เจอแล้วเกิด error ซึ่ง error ตัวนี้มันไม่
ค่อยเกี่ยวข้องอะไรกับ business logic ของเรา ดูๆไปเราสามารถ ignor ได้ แทนที่จะ
